## 新增功能

### 1. 都察院审核机制 ✅

**功能描述：** 当任务类型涉及"代码"、"开发"、"文章"等关键词时，任务执行后需要经都察院审核。

**检测关键词：**
- 代码、编程、开发、code
- 文章、文档、报告
- 撰写、编写、创作
- 代码审查

**审核流程：**
```
部门执行完成 → 都察院移动到该部门 → 审核中 → 审核通过 → 都察院返回 → 继续汇报链
```

**视觉反馈：**
- 对话框中输入相关命令时自动显示提示 🛡
- 都察院角色使用紫色（`#aa66ff`）发光效果
- 审核过程中显示"🛡 审核中"状态

**防止自审循环：** 都察院自己的任务不会触发自我审核。

---

### 2. 失败重试上报机制 ✅

**功能描述：** 任务执行失败后自动重试（最多3次），重试耗尽后逐级上报失败原因给陛下。

**重试逻辑：**
```
执行失败 → 重试(1/3) → 失败 → 重试(2/3) → 失败 → 重试(3/3) → 失败 → 上报失败原因
```

**失败原因列表（随机）：**
- 资源不足，缺少关键依赖
- 网络连接超时
- 权限校验未通过
- 服务暂不可用
- 数据格式异常

**上报链（完整）：**
```
部门 → 上级领导 → 冥王 → 陛下（接收失败原因）
```

**视觉反馈：**
- 失败时角色显示红色闪烁动画（`error-flash`）
- 调度日志显示"❌ 失败(N/M)"和失败原因
- 底部横幅显示"🚨 失败上报完成"

---

### 3. 完整汇报链 ✅

**功能描述：** 确保多级部门汇报时逐级上报到陛下，形成完整的汇报闭环。

**修复前（六部流程）：**
```
部门 → 尚书令 → 冥王 → 冥王回位（❌ 没有到陛下）
```

**修复后（六部流程）：**
```
部门执行 → [都察院审核] → 尚书令汇报 → 冥王汇报 → 陛下接收 → 冥王回位
```

**修复前（内阁/内廷下属流程）：**
```
部门 → 上级领导 → 冥王 → 冥王回位（❌ 没有到陛下）
```

**修复后（内阁/内廷下属流程）：**
```
部门执行 → [都察院审核] → 上级汇报 → 冥王汇报 → 陛下接收 → 冥王回位
```

**修复前（直属部门流程）：**
```
部门 → 冥王 → 冥王回位（❌ 没有到陛下）
```

**修复后（直属部门流程）：**
```
部门执行 → [都察院审核] → 冥王汇报 → 陛下接收 → 冥王回位
```

## 技术实现细节

### 新增函数
- `needsDuchaReview(command, skipDept)` - 检测任务是否需要都察院审核
- `runDuchaReview(targetDept, command, callback)` - 都察院审核动画流程
- `executeWithRetry(dept, command, onAttempt, onFailure, onSuccess)` - 重试执行机制

### 新增常量
- `MAX_RETRY = 3` - 最大重试次数

### FSM 修改
- `_runSixMinistry(dept, command, requiresReview)` - 六部调度链（含审核+重试）
- `_runWithParent(dept, command, parent, requiresReview)` - 内阁/内廷下属调度链
- `_runDirectDept(dept, command, requiresReview)` - 直属部门调度链
- `_runLiubuShangshulingSelf(dept, command)` - 六部尚书令本人调度链

### 新增 CSS 动画
- `receiving-pulse` - 领旨/接收状态（蓝紫光晕脉动）
- `reporting-glow` - 汇报状态（金色光晕脉动）

### 一键演示
- 现在使用"编写代码"命令，可同时演示都察院审核+失败重试机制
