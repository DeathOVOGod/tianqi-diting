# Gomoku AI - 五子棋 AI

**文件**: `gomoku_ai.py`
**规则**: 15×15 棋盘，无禁手（自由五子棋）
**核心算法**: Alpha-Beta + Transposition Table + Zobrist Hash + Null Window + Aspiration Windows + PVS
**版本**: v3 (基于翰林院深度研究报告的精细化优化版)

---

## 快速开始

### 1. 命令行对局（人 vs AI）
```bash
python3 gomoku_ai.py
```
- 执黑先手 (X)，AI 执白 (O)
- 输入格式: `row col` (如 `7 7`)
- 输入 `quit` 退出

### 2. 程序化调用
```python
from gomoku_ai import get_best_move

board = [[0]*15 for _ in range(15)]
board[7][7] = 1   # 黑棋落子 (天元)
board[7][8] = 2   # 白棋落子

move = get_best_move(board, player=2, depth=30, time_limit=10.0)
# 返回: (row, col) 或 None
print(f"AI 建议: {move}")
```

### 3. 批量查询（管道输入）
```bash
cat positions.txt | python3 gomoku_ai.py --batch
```
格式: 每行 `row,col row,col ... | player`
示例: `7,7 8,8 7,8 8,9 | 1`

### 4. 评估模式
```bash
python3 gomoku_ai.py --eval
```
测试AI对必胜开局的应对能力。

## v3 升级内容（基于翰林院深度研究报告）

### 1. 评估函数精细化重构
- **评分梯度拉开**：五连10亿 > 活四10万 > 冲四1万 > 双三8000 > 活三1000 > 眠三200
- **位置权重矩阵**：15×15预计算位置权重，中心价值高，边缘低
- **防守加权强化**：对手冲四防守+THREAT_DEFENSE_BONUS=2.5倍，对手活三防守+防守加权5000
- **威胁防御评分**：`evaluate_candidate`中加入对手威胁位置检测

### 2. 置换表升级（固定大小数组）
- **数组替代dict**：8M条目固定大小数组，替代Python dict，内存效率提升3-5倍
- **Secondary Probe**：哈希冲突二次探测
- **深度优先替换策略**：新条目深度必须≥旧条目才覆盖
- **8位age循环**：避免条目无限增长

### 3. 开局库扩展（前10步，50+定式）
- 收录天元开局所有常见变化（前10步）
- 覆盖进攻型/平衡型/防守型三种开局风格
- 花月/寒星/浦月等经典开局定式
- 自动识别开局序列并返回最优应对

### 4. 杀手启发增强
- **TopK排序**：每个深度只保留2个杀手走法，替代FIFO
- **按优先级排序**：新条目插入到前面
- 避免重复添加相同走法

### 5. 历史启发衰减
- **衰减因子0.9**：每轮搜索后历史启发值衰减，防止溢出
- **16位整数存储**：避免Python整数开销

### 6. 空步启发动态化
- **动态空步位置**：选择最后一手位置作为虚拟空步，替代硬编码天元(7,7)
- 启发更加精准

### 7. 动态 Aspiration Window
- **初始窗口3000**：替代固定±5000
- **渐进膨胀**：搜索失败时窗口翻倍（最大50000）
- 减少重新搜索次数

### 8. PVS 完整实现
- **第一个节点全窗口**：基准搜索
- **后续节点零窗口**：加速剪枝
- **零窗口失败重搜**：如果结果在(α,β)之间，重新用全窗口

### 9. 防御逻辑强化
- **`should_defend()`**：检测对手活四/冲四/活三威胁
- **`get_defensive_move()`**：强制返回防守走法
- **`get_best_move()`**：搜索前先检查是否需要强制防守
- 冲四必须立即防守，活三高度优先

### 10. 时间控制优化
- **MAX_SEARCH_TIME = 10秒**：替代12秒（更合理）
- **渐进式时间分配**：开局/中局/残局动态分配
- **80%时间上限**：每轮迭代加深最多用80%时间

### 11. 走法数量动态调整
- 根节点：min_range=1，候选点更多
- 深度搜索：最多25个走法（替代30个），防止分支爆炸

## 架构设计

### 搜索算法
- **Alpha-Beta 剪枝**：O(b^(d/2)) 复杂度
- **迭代加深**：偶数层迭代，稳定给出当前最优解
- **Null Window Search**：零窗口搜索加速
- **Principal Variation Search (PVS)**：完整PVS实现
- **置换表（TT）**：8M条目固定数组，二次探测
- **Zobrist Hash**：64位哈希
- **杀手启发**：TopK排序（每深度2个）
- **历史启发**：衰减因子0.9
- **空步启发**：R=2，动态空步位置

### 评估函数
| 棋型 | v2分数 | v3分数 | 变化 |
|------|--------|--------|------|
| 五连 | 10,000,000 | 1,000,000,000 | x100 |
| 活四 | 1,000,000 | 100,000 | ÷10 |
| 冲四 | 200,000 | 10,000 | ÷20 |
| 双三 | N/A | 8,000 | 新增 |
| 活三 | 50,000 | 1,000 | ÷50 |
| 眠三 | 5,000 | 200 | ÷25 |
| 活二 | 800 | 100 | ÷8 |
| 眠二 | 100 | 20 | ÷5 |

### 走法生成
- 只搜索**有意义的点**：以已有棋子为中心，半径 2-3 格范围内的空位
- 走法数量上限25个（深度搜索）
- 走法排序：TT走法 > 杀手走法 > 历史启发 > 评估分

## 必胜策略（无禁手规则）

自由五子棋（无禁手）已被数学证明：**先手（执黑）必胜**。

本 AI v3 在执黑时会：
1. 第一步走天元（中心）
2. 第二步走斜向位置，形成进攻态势
3. 通过30层深度 Alpha-Beta 搜索找到必胜路径
4. 优先形成活四/冲四威胁，对对手威胁高度敏感
5. 强制防守对手的冲四/活三威胁
